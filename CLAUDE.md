
# Mathbook Project Instructions

## Golden Reference

Chapter 4 on the `main` branch is the canonical implementation for this project.

All future chapters and lessons must match Chapter 4 in teaching style, visual design, interaction patterns, and code quality.

---

## Teaching Style

- Mathematics-first.
- Keep explanations concise.
- Show complete step-by-step derivations.
- Use arrow-connected equation chains.
- Box important intermediate and final results.
- Preserve mathematical correctness.

---

## Design Rules

Reuse existing project components whenever possible.

Keep the same:

- HTML structure
- CSS classes
- JavaScript patterns
- SVG style
- Canvas style
- Interactive widgets
- Typography
- Layout
- Responsive behavior

Never introduce a different visual style.

---

## Workflow

For every lesson:

1. Analyze the equivalent implementation in Chapter 4.
2. Refactor the lesson.
3. Verify consistency.
4. Run Playwright.
5. Fix every issue.
6. Commit changes.
7. Continue to the next lesson.

Work lesson-by-lesson.

Never refactor multiple lessons simultaneously unless requested.
